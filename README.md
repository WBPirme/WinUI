# OGAS

OGAS 是一个基于 `WPF + .NET 8 + EF Core + SQL Server LocalDB` 构建的桌面端生产资源管理与计划分配系统。这个项目围绕生产型业务中的核心环节展开，将基础资料维护、生产计划管理、生产执行、库存监控、出口订单以及首页数据看板整合到同一个应用中，形成了一条较完整的业务流程链路。

它不是一个单纯的“录数据工具”，而是一个面向生产管理场景的综合业务系统原型。项目希望解决的问题，是如何把生产相关的资源、规则、计划、执行结果和经营数据统一放进一个桌面应用中进行管理，让用户能够从“配置资源”一路走到“查看产出与收益”。

## 项目简介

从业务定位上看，OGAS 可以理解为一个小型生产运营管理平台的桌面版。系统把生产活动中最常见的几个环节串在了一起：

`基础数据维护 -> 生产计划制定 -> 自动生产处理 -> 库存变动 -> 出口结算 -> 数据总览`

项目中的数据和页面并不是彼此孤立的，而是围绕同一条生产业务主线来设计的：

- 先定义产品、材料、设备、工厂和工艺这些基础对象。
- 再根据工艺与工厂能力创建生产计划。
- 然后通过生产处理模块推进任务执行。
- 执行完成后自动生成生产记录，并同步更新库存。
- 最后通过出口订单模块完成出库和收益计算。
- 所有关键结果最终汇总到首页，用于展示计划状态、库存状态、收益和运行信息。

因此，这个项目更准确的定位不是“仓库系统”或“简单计划表”，而是一套覆盖生产前、中、后多个阶段的桌面端业务管理系统。

## 这个项目主要做了什么

如果用较容易理解的话来概括，这个项目主要完成了下面几件事：

- 管理生产场景中的核心主数据，包括产品、材料、加工厂、生产设备和生产工艺。
- 根据工艺与工厂能力创建生产计划，并对计划分配过程做基本业务约束。
- 用自动生产逻辑来模拟或驱动生产任务执行，计算加工时间并展示进度。
- 让生产执行结果自动反映到材料库存、产品库存和生产记录中。
- 支持出口订单创建，在出库时校验库存并自动计算收益。
- 提供首页总览界面，将计划、库存、收益、日志和图表信息汇总展示。

这意味着 OGAS 的重点并不只是“维护一张表”，而是在尝试表达一套有内在关系的生产管理业务逻辑。

## 核心业务对象

项目中的核心业务对象大致包括以下几类：

### 1. 产品

产品是系统最终要生产、入库、统计和出口的对象。项目中为产品维护了名称、类别、描述、计量单位和单价等信息。产品既会参与生产工艺定义，也会关联产品库存和出口订单。

### 2. 材料

材料代表生产过程中需要消耗的原料。它和产品类似，也包含基础属性，并会在生产工艺中被引用，在生产完成时参与库存扣减。

### 3. 生产设备

生产设备是工艺执行所依赖的能力资源。项目中为设备定义了容量和效率等属性，这些参数会直接参与生产时间与产出能力的计算。

### 4. 加工厂

加工厂代表任务实际执行的生产单位。工厂会绑定设备，并具有设备数量、位置、联系人、加工效率等信息。生产计划在创建时并不是随意分配工厂，而是需要匹配工艺所需设备。

### 5. 生产工艺

生产工艺是整个系统里非常关键的业务实体。它定义了：

- 生产哪个产品
- 是否需要消耗某种材料
- 依赖哪种生产设备
- 标准加工时间是多少
- 质量标准、安全指引和备注信息是什么

可以把工艺理解成“生产规则模板”，后续的生产计划和生产执行，都是围绕工艺展开的。

### 6. 生产计划

生产计划用于把工艺、工厂、数量、日期和状态组合成一个实际任务。系统中的计划不是静态记录，而是会进入后续处理流程，被自动生产模块读取和推进。

### 7. 生产记录

生产记录是在任务执行完成后产生的结果数据，用来记录实际执行情况。它是计划落地后的痕迹，也是库存变化与统计分析的重要依据。

### 8. 库存

库存分为产品库存和材料库存。它们分别用于表示产出结果和原料剩余情况。生产完成时会产生库存变动，出口订单创建时也会触发产品库存扣减。

### 9. 出口订单

出口订单表示产品对外输出或销售的业务记录。创建订单时系统会检查库存是否充足，并按照产品单价和出口数量计算收益。

## 系统功能模块

### 1. 登录模块

应用启动后首先进入登录窗口。用户需要输入账号和密码，只有通过数据库中的用户信息校验后，才能进入主窗口。这个设计说明项目已经把“身份入口”纳入完整流程，而不是直接展示页面。

对于一个桌面管理系统来说，登录模块虽然不复杂，但它强化了系统作为“正式业务应用”的形态。

### 2. 首页主控台

首页承担的是运营看板和状态总览的角色，不是主要的数据编辑页面。它集中展示了多个模块的结果信息，例如：

- 计划状态数量和占比
- 材料库存与产品库存总数
- 出口总量与出口收益
- 系统日志
- 网络流量图表

这使得用户在进入系统后，可以先看到整体情况，再决定进入哪个业务模块做具体操作。

### 3. 数据采集 / 基础资料管理模块

这是系统所有后续业务的基础。该模块负责维护：

- 产品
- 材料
- 加工厂
- 生产工艺
- 生产设备

从页面设计和 ViewModel 逻辑来看，这一模块支持典型的增删改查操作，并且不仅仅是单表录入，还带有一定的关联约束。例如：

- 加工厂需要关联生产设备。
- 生产工艺需要关联产品、材料和设备。
- 删除某些实体时会检查是否仍被其他业务对象引用。

这说明项目在基础数据层面已经考虑到了业务关系完整性，而不是简单地做表格编辑。

### 4. 生产计划管理模块

这个模块负责把基础数据组织成待执行任务。用户可以在此管理生产计划，包括：

- 计划类型
- 关联工艺
- 执行工厂
- 计划数量
- 开始日期
- 状态
- 备注

这个模块最有价值的地方在于，它不是简单保存一条计划记录，而是加入了“可执行性”判断：

- 选择工艺后，系统会筛选出满足设备条件的加工厂。
- 如果工厂设备与工艺要求不匹配，计划就不能合理执行。

这让计划模块带有一定的资源调度逻辑，而不是只负责填写计划表。

### 5. 生产处理模块

生产处理模块是整个项目最接近“执行层”的部分。它的作用不是维护静态数据，而是推进生产任务运行。

该模块支持：

- 启用自动生产
- 遍历未完成的生产计划
- 根据工艺时间、设备容量、设备效率、工厂效率、设备数量等参数计算总生产时间
- 展示任务进度条和剩余时间
- 标记等待中的任务
- 暂停与恢复生产

从实现逻辑上看，项目尝试模拟一个简化版的生产调度过程：

- 如果工厂忙碌，任务进入等待状态。
- 如果工厂空闲，则开始执行对应计划。
- 任务完成后写入生产记录，并更新计划状态和库存。

这部分逻辑让系统从普通 CRUD 应用上升到了“带业务流程推进”的管理系统。

### 6. 库存与监控模块

该模块主要覆盖两个方面：

### 库存管理

包括产品库存和材料库存的维护，支持：

- 新增库存记录
- 更新库存记录
- 删除库存记录
- 根据不同类型切换产品库存和材料库存管理

### 出口订单管理

包括：

- 选择出口产品
- 填写出口数量
- 自动计算订单收益
- 校验库存是否充足
- 创建订单后同步减少产品库存

这个模块让系统不仅能“生产”，还能把生产结果延伸到“出库与收益”环节，因此业务闭环更完整。

## 业务流程怎么串起来

这个项目最值得介绍的一点，是各模块之间是有顺序关系的，而不是各做各的。

### 第一步：先配置基础资源

录入产品、材料、设备、工厂和工艺。没有这些基础对象，就无法谈后续的计划和执行。

### 第二步：再创建生产计划

根据工艺和工厂资源情况创建任务，明确生产什么、在哪里生产、生产多少以及从什么时候开始。

### 第三步：启动生产处理

生产处理模块读取未完成计划，结合设备能力和工厂效率计算执行时间，并用进度条显示生产状态。

### 第四步：完成后更新库存与记录

当任务完成后，系统会自动：

- 扣减材料库存
- 增加产品库存
- 记录生产数据
- 更新计划状态

### 第五步：创建出口订单

当成品需要对外输出时，系统会检查库存是否足够，然后生成订单并计算收益。

### 第六步：首页展示整体结果

最终，计划情况、库存情况、收益情况以及系统日志都会在首页形成集中展示。

这一整套流程说明，项目虽然规模不大，但已经具有“端到端业务联动”的雏形。

## 这个项目的亮点

如果站在“给别人介绍项目”的角度，OGAS 的亮点主要体现在以下几点：

- 不是单页面或单功能，而是覆盖多个业务模块。
- 不是纯录入系统，而是带有生产计划分配和执行推进逻辑。
- 数据之间存在明确关系，体现了较完整的业务建模思路。
- 生产完成会自动影响库存，出口订单会自动影响收益和出库。
- 首页看板把多个模块的数据结果汇总展示出来，具备较好的整体感。
- 桌面端采用 `WPF + MVVM`，后端数据处理采用 `EF Core`，结构相对清晰。

如果把它作为一个作品来介绍，这个项目的优势不在于单点炫技，而在于“模块之间是联动的”。

## 技术实现

项目主要采用了以下技术栈：

- 桌面框架：WPF
- 运行平台：`.NET 8` (`net8.0-windows`)
- 架构模式：MVVM
- 数据访问：Entity Framework Core
- 数据库：SQL Server LocalDB
- 依赖注入与应用宿主：`Microsoft.Extensions.Hosting`
- 图表展示：OxyPlot
- 日志组件：NLog

从代码组织方式来看，目录职责也比较清晰：

- `Views`：界面页面
- `ViewModels`：页面逻辑与业务处理
- `Models`：核心业务实体
- `Data`：数据库上下文与数据访问入口
- `Services`：导航等通用服务
- `Controls`：自定义控件
- `Converters`：数据绑定转换器

## 总结

综合现有代码来看，OGAS 做的是一个偏实验性、但结构完整度较高的生产管理类桌面应用。它不是只处理一个环节，而是尽量把生产业务中的几个关键阶段连接起来：

- 前端有登录、看板、表格、表单、图表和进度展示。
- 逻辑层有计划分配、库存校验、生产推进和收益计算。
- 数据层有实体关系、生产记录、库存记录和订单记录。

因此，这个项目最适合被定义为：

一个面向生产场景的桌面端综合管理系统原型，用于统一管理资源、计划、生产执行、库存变化和出口收益。
